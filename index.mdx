High-Energy gives you a complete pipeline for parallel ROOT TTree analysis — from reading raw Monte Carlo data through histogram filling to polished plot output. Configure every stage with TOML, wire in your analysis callback, and let the framework handle threading, checkpointing, and progress reporting.

## What you can build

High-Energy is designed for physicists and analysts who need to process large ROOT TTree datasets — millions of events — in parallel, accumulate histograms thread-safely, and render publication-quality plots without writing boilerplate threading code.

## Getting started

<Steps>
  <Step title="Install dependencies">
    Install ROOT 6, Pythia8, and toml++ on your system. See the [Installation guide](/installation) for package manager commands and version requirements.
  </Step>
  <Step title="Configure your analysis">
    Copy `configs/Lambda_Reconstruction.toml` as a starting point and set your input file path, thread counts, and output directory.
  </Step>
  <Step title="Write your callback">
    Implement an analysis function with signature `void(const Probe::Event&, int threadId)` and register histograms with `Record::Writer`.
  </Step>
  <Step title="Build and run">
    Run `make _Lambda_Reconstruction` and execute the binary. The Monitor prints live progress, saves checkpoints, and writes the final ROOT output file.
  </Step>
</Steps>
